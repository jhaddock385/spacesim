# Handoff: 2026-03-30

## What This Project Is

A Star Trek-style bridge sim where the player is the captain and issues natural language commands to AI crew members (Claude API). Crew members interpret commands through their personality and skill level, then take actions against a deterministic ship simulation. The AI is the interface — you can't click buttons to fire torpedoes, you have to tell your tactical officer.

## Architecture Summary

**Three-layer design** (only layers 1 and 2 are partially built):
1. **Simulation** (`src/sim/`) — deterministic, pure Lua, no AI. Event bus + state store.
2. **AI Agents** (`src/agents/`) — Claude API with tool use. Prompt builder assembles identity + knowledge + state + tools.
3. **Interface** (`src/screens/`) — Love2D rendering. Simulator screen has direct control and comms modes.

**Everything flows through the event bus.** UI clicks, AI crew actions, and internal system events all emit onto the same bus. No subsystem directly mutates another's state. The bus drains once per physics tick with a depth limit of 20.

**State lives in the partitioned store.** Keyed by entityId/component/key. All systems read from and write to the store.

**Fixed timestep at 60Hz.** Clock accumulates dt, steps in fixed increments. Freeze/unfreeze for pausing.

**Async API calls.** Curl runs in the background, game polls each frame. No UI blocking.

## What Works Well

- The event bus architecture is clean and extensible
- The AI crew member (Kim) correctly interprets commands, uses tools, and executes multi-step actions
- Tool use continuation works — Kim can query instruments, get results, then formulate verbal responses
- Conversation history gives multi-turn context within a session
- The pip-based power model creates real tradeoffs
- Async API keeps the UI responsive

## Known Issues / Technical Debt

- `simulator.lua` is ~500 lines and does too many jobs (spatial map, system panel, event log, comms). Needs splitting into submodules.
- The JSON encoder needed a special `ARRAY_MARKER` sentinel to distinguish empty arrays from empty objects. Works but is a bit ugly.
- No conversation memory across sessions — history resets when you leave the simulator screen.
- Kim can't actually hold a heading — she can start a turn but has no feedback loop to stop at a target heading. This needs either a more sophisticated action (setHeading with a target) or a multi-turn approach where she monitors and adjusts.
- The `secrets.lua` file must exist locally with `anthropic_api_key`. If it's missing, comms mode will show an error.

## Design Principles (important — read these)

From `CLAUDE.md`, enforced by the owner:

- **Think before building.** Discuss tradeoffs before implementing. Don't race to features.
- **Event bus everywhere.** All state changes through the bus, at every scale. No shortcuts.
- **No parallel abstractions.** Check if an existing system covers the concern before creating new ones.
- **Read architecture docs before modifying existing systems.**
- **No commits without being asked.** No `git add .`. No AI attribution in commits.
